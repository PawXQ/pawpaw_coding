2024-06-09
C# article

Contents

以下會介紹兩種"nested if-else statements"的寫法.

題目很簡單就是Pizza好不好吃的判斷,有大量的"if-else"判斷, 會有許多的判斷是崁套在code裡面

條件：

需要有判斷輸入的文字是否為數字

波動拳(正向列舉)

從下方可以看到code會一層一層往內縮, 最後會在中間出現當所有condition皆正確時會執行的動作

這種方式雖然容易編寫, 但在未來會不容易維護

在一般寫有大量 if else 判斷式的過程中很容易出現這種現象, 因此會介紹第二種方式

string tem_str_1 = Console.ReadLine(); // 請user輸入數字
if (int.TryParse(tem_str_1, out int tem_int_1)) // 判斷user的輸入能否轉換成數字
{
    if (tem_int_1 >= 120 && tem_int_1 <= 125) // 判斷input是否在 120~125 之間
    {
        string wet_str_1 = Console.ReadLine(); // 條件成立狀況下, user輸入第二的數字
        if (int.TryParse(wet_str_1, out int wet_int_1)) // 判斷user的輸入能否轉換成數字
        {
            if (wet_int_1 >= 5 && wet_int_1 <= 7) // 判斷input是否在 5~7 之間
            {
                Console.WriteLine("good!"); // 條件都成立, 返回 good! 的文字
            }
            else
            {
                Console.WriteLine("bad!"); // 條件不成立, 返回 bad! 的文字
            }
        }
        else
        {
            Console.WriteLine("error syntax!"); // user 輸入的文字不能轉換成數字, 返回 error syntax
        }
    }
    else
    {
        Console.WriteLine("bad!"); // 條件不成立, 返回 bad! 的文字
    }
}
else
{
    Console.WriteLine("error syntax!"); // user 輸入的文字不能轉換成數字, 返回 error syntax
}

條件判斷式(反向列舉)

此種方式就不會出現, 砍套式一層一層內縮的狀況

此種方式為按照判斷順序, 將所有的錯誤條件列出, 在一小段code中就可以知道判斷內容

因此在維護過程中, 就不需要找到最中間的部分, 因為可以知道當所有條件成立時

所要執行的動作一定在最下方
